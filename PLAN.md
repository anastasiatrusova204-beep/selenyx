# План реализации MVP — Selenyx

> Код пишет Claude. Человек читает, запускает и проверяет результат.
> Каждый шаг = видимый результат. После каждого шага — проверка.

---

## Шаг 1 — Запуск: бот отвечает на /start
**Результат:** открываешь Telegram, пишешь боту `/start` — он отвечает.

Что делаем:
- Создаём структуру проекта (папки и файлы)
- Устанавливаем aiogram (библиотека для Telegram-ботов)
- Пишем минимальный бот: только `/start`
- Запускаем локально

Проверка: бот отвечает приветствием в Telegram.

---

## Шаг 2 — Онбординг: выбор знака зодиака
**Результат:** при старте бот спрашивает знак, запоминает — и при следующем входе уже знает кто ты.

Что делаем:
- Добавляем inline-кнопки с 12 знаками зодиака
- Сохраняем выбор в SQLite (встроенная БД Python, ничего устанавливать не нужно)
- Бот подтверждает: «Запомнил — ты Овен»

Проверка: выбираем знак, перезапускаем бота, снова пишем `/start` — бот помнит знак.

---

## Шаг 3 — Луна: текущее положение
**Результат:** команда `/moon` показывает реальную фазу и знак Луны прямо сейчас.

Что делаем:
- Устанавливаем kerykeion (астрологическая библиотека, расчёты по NASA)
- Пишем функцию `get_moon_data()` — возвращает фазу и знак
- Добавляем обработчик команды `/moon`

Проверка: `/moon` выдаёт реальные данные — сверяем с любым лунным календарём.

---

## Шаг 4 — Энергия дня: /today
**Результат:** команда `/today` показывает полную подсказку на сегодня.

Что делаем:
- Пишем шаблоны текстов для каждой фазы Луны (8 фаз)
- Функция `get_daily_energy()` собирает подсказку из астро-данных + шаблона
- Добавляем обработчик `/today`

Проверка: `/today` показывает разный текст в разные дни / фазы Луны.

---

## Шаг 5 — Главное меню: кнопки вместо команд
**Результат:** пользователь не вводит команды вручную — нажимает кнопки.

Что делаем:
- Добавляем ReplyKeyboard (кнопки внизу экрана) с тремя пунктами
- Форматируем сообщения: эмодзи, отступы, читаемый вид
- Кнопка «О боте» показывает описание и дисклеймер

Проверка: бот полностью управляется кнопками, выглядит как готовый продукт.

---

## Шаг 6 — Деплой: бот работает 24/7
**Результат:** бот запущен в облаке — работает без твоего компьютера.

Что делаем:
- Создаём `Procfile` и `railway.toml` для Railway
- Переносим секреты в переменные окружения
- Деплоим через GitHub или CLI
- Проверяем, что бот отвечает без локального запуска

Проверка: выключаем компьютер, пишем боту — он отвечает.

---

```
│   ├── services/

├── config.py              # настройки из .env
├── requirements.txt       # зависимости
├── .env.example           # шаблон переменных окружения
├── Procfile               # для Railway (шаг 6)

├── RESEARCH.md

```
