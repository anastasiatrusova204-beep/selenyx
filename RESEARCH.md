## 3. UX И СЦЕНАРИИ

### Сценарий 1: Утренний ритуал
**Профиль:** женщина, 28 лет, заходит каждое утро.

1. Видит уведомление в 8:00.
2. Открывает: «Луна в Деве, день для деталей. Энергия дня — 7/10.»
3. Читает 2–3 рекомендации (что начать, что отложить).
4. Делится скрином в Stories.

**Что важно:** читается за 30 секунд, ощущение персонализации.

---

### Сценарий 2: Ситуативная проверка
**Профиль:** мужчина, 35 лет, заходит по конкретному поводу.

1. Перед важными переговорами открывает бот.
2. Пишет `/today`, получает: фаза Луны, энергия, совет по коммуникациям.
3. Принимает решение — проводить встречу или перенести.
4. Уходит, возвращается через 2–3 дня.

**Что важно:** минимум действий, мгновенный ответ, конкретный совет.

---

### Сценарий 3: Онбординг нового пользователя
**Профиль:** новый пользователь, 22 года, пришёл по рекомендации.

1. Переходит по ссылке, видит приветствие.
2. Бот задаёт 2–3 вопроса: знак зодиака или дата рождения.
3. Получает первый прогноз сразу — без регистрации.
4. Видит предложение подписаться на уведомления.

**Что важно:** первая ценность в течение 60 секунд, онбординг без перегрузки.

---

### Главные раздражители пользователей (из отзывов)

- **Агрессивный paywall** — платить за каждую функцию поверх подписки (жалоба №1 в Moonly и Co-Star).
- **Слишком много уведомлений** — 62% раздражают частые пуши. Оптимум: 1 сообщение в день.
- **Общие тексты** — «гороскоп для Овнов» без персонализации воспринимается как мусор.
- **Непрозрачная цена** — узнать о платном контенте после регистрации злит.
- **Медленный ответ** — ожидание > 3 секунд разрушает «магию момента».

## 4. ТЕХНОЛОГИИ

### Kerykeion — рекомендуется для MVP

Астрологическая Python-библиотека. Под капотом — Swiss Ephemeris (NASA JPL), точность ~1 угловая секунда.

**Что умеет:** позиции планет и домов, аспекты, натальная карта, синастрия, транзиты, фазы Луны с % освещённости и знаком, генерация SVG-чартов.

- **Плюсы:** активная поддержка (обновления 2024–2025), современный Python API (3.9+), структурированный вывод данных, хорошая документация.
- **Минусы:** лицензия AGPL-3.0 — для коммерческого использования нужна лицензия Swiss Ephemeris (~$250–1000 единоразово).

**Вердикт:** лучший выбор — полный функционал + живая поддержка.

---

### ephem (PyEphem) — альтернатива для минимального MVP

Астрономическая библиотека, не астрологическая. MIT-лицензия, нет ограничений на коммерцию.

**Что умеет:** позиции Луны и планет, фазы Луны, восходы/закаты. Знаки зодиака и домa — нужно дописывать вручную.

- **Плюсы:** бесплатно навсегда, стабильная, быстрая.
- **Минусы:** не астрологическая — придётся много писать самостоятельно. Нет натальной карты.

**Вердикт:** подходит если нужна только Луна и не планируется натальная карта.

---

### flatlib — не рекомендуется

Астрологическая библиотека. Swiss Ephemeris под капотом.

- **Минусы:** заброшена с 2018–2019 года, открытые issues без ответа, проблемы совместимости с Python 3.10+.

**Вердикт:** высокий технический риск — не использовать.

---

### Итоговое сравнение

| Критерий | Kerykeion | ephem | flatlib |
|----------|-----------|-------|---------|
| Натальная карта | Да | Нет | Да |

| Активная поддержка | Да | Да | Нет |

| Рекомендация | **MVP** | Если только Луна | Не использовать |

## 6. TELEGRAM MINI APP В НИШЕ АСТРОЛОГИИ И ЭЗОТЕРИКИ

### Конкуренты и Mini App: что уже есть на рынке

На момент написания (март 2026) Mini App в астро-нише используют единицы:

| Продукт | Есть Mini App? | Как используют |
|---|---|---|
| Saturn.Love | Нет | Только бот |
| Numia | Нет | Только бот |
| Co-Star | Нет (iOS/Android) | Нативное приложение |
| Moonly | Нет (iOS/Android) | Нативное приложение |
| The Pattern | Нет (iOS/Android) | Нативное приложение |
| AstroBot (мелкие боты) | Частично | Лендинг под Stars-покупки |

**Вывод:** ниша пустая. Ни один из прямых конкурентов в Telegram не реализовал полноценный Mini App с контентом. Это окно возможностей — и одновременно зона неизведанного.

---

### Почему Mini App особенно важен именно в астро-нише

**1. Эмоциональный продукт требует визуальной среды**
Астрология продаёт ощущение — связи с ритмом, особенности, магии момента. Текстовый чат-бот передаёт это слабо. Mini App с анимацией, тёмной палитрой и красивой типографикой создаёт нужную атмосферу.

**2. Информация плотная — чат не справляется**
Фаза Луны + знак + лунный день + аспекты + ретрограды + цвет дня + нумерология — это 7+ единиц информации. В боте это 7 сообщений или одно длинное нечитаемое. В Mini App — один красивый экран с вкладками.

**3. Ежедневный ритуал нуждается в «месте»**
Пользователи Coffee приложений не пьют кофе в командной строке. Аналогично: ежедневная астро-практика нуждается в «пространстве» — интерфейсе, в который приятно возвращаться. Бот — инструмент. Mini App — место.

**4. Монетизация через Stars работает лучше визуально**
«Разблокировать за ⭐ 490» с blur-эффектом видимого контента конвертирует выше, чем inline-кнопка в сообщении. Пользователь уже видит ценность — осталось разблокировать.

---

### Критика Telegram Mini App: позиция разработчиков и экспертов

#### Технические ограничения (реальные проблемы в production)

**1. initData истекает через 24 часа**
Telegram подписывает initData timestamp-ом. Через 24 часа токен устаревает. Если пользователь держит Mini App открытым долго или Telegram кэширует страницу — API начнёт отдавать 401.
*Решение:* на стороне API игнорировать проверку времени жизни или добавить refresh-механизм.

**2. Нет DevTools в production**
В настоящем Telegram-клиенте нет инструментов разработчика. Ошибки в JS видны только если добавить свой error overlay или Telegram BotAPI debug mode.
*Решение:* `@tg_miniapp_test_bot` для тестирования, `window.onerror` → показывать сообщение об ошибке в UI.

**3. Непредсказуемое кэширование**
Telegram Desktop агрессивно кэширует Mini App. iOS-клиент — меньше. Изменения в index.html могут не применяться у части пользователей до принудительного обновления.
*Решение:* добавить query-параметр версии (`?v=2`) к URL при деплое.

**4. Шрифты из Google Fonts CDN — риск задержки**
При первом открытии без кэша браузер делает запросы к CDN. Если пользователь в зоне с плохим интернетом — FOUT (Flash of Unstyled Text) или медленный рендер.
*Решение:* `<link rel="preconnect">` + `font-display: swap` + fallback-шрифт схожей геометрии.

**5. Back Button iOS — неожиданное поведение**
На iOS системная кнопка «назад» иногда закрывает Mini App вместо навигации внутри SPA. Нужно явно обрабатывать `tg.BackButton`.
*Решение:* синхронизировать `tg.BackButton.show()/hide()` с состоянием навигации приложения.

**6. Высота экрана непредсказуема**
`window.innerHeight` меняется при появлении клавиатуры, системных панелей, при переходе в fullscreen. Нижний таб-бар может «ехать».
*Решение:* `tg.expand()` при старте + `env(safe-area-inset-bottom)` для таб-бара.

#### UX-критика (от продукт-дизайнеров)

**«Пользователь не знает что открывает»**
Кнопка Menu в боте выглядит как обычная кнопка Telegram — пользователь не ожидает, что откроется полноценное приложение. Первые 2 секунды — момент недоумения.
*Решение:* анимация входа (сплэш + нарастание) сигнализирует: «это не бот, это приложение».

**«Нет оффлайн-режима»**
При плохом интернете Mini App показывает пустой экран или ошибку. Нативные приложения (Co-Star, Moonly) работают с кэшем.
*Решение:* Service Workers в Telegram Mini App не поддерживаются. Показывать skeleton-screens и graceful error states.

**«Потеря контекста при сворачивании»**
Если пользователь свернул Telegram и вернулся через 10+ минут — Mini App может перезагрузиться. Вкладки и состояние не сохраняются.
*Решение:* минимальное состояние в `sessionStorage`, восстанавливать активную вкладку.

#### Коммерческая критика

**Telegram Stars комиссия**
Telegram берёт 30% с каждой Stars-транзакции. В App Store тоже 30%. Нет двойной комиссии — но и не «бесплатно».

**Вывод средств — только через Fragment в TON**
Деньги приходят в Toncoin, не в рубли напрямую. Нужна конвертация, что добавляет комплексность для небольшого бизнеса.

**«Telegram может изменить политику»**
Mini App — продукт внутри чужой платформы. Telegram может изменить правила, поднять комиссию или закрыть API. Стратегический риск для бизнеса без собственного приложения.

### Что такое Telegram Mini App

Веб-приложение (HTML/CSS/JS), открывающееся прямо внутри Telegram. Пользователь не выходит из мессенджера, не скачивает отдельное приложение. Доступ через кнопку Menu в чате бота или inline-кнопку.

**Ключевое отличие от бота:** вся логика в браузерном интерфейсе, а не в цепочках сообщений. Пользователь управляет приложением тапами, а не командами.

---

### Почему Mini App — это правильный шаг

| Критерий | Telegram-бот (только) | Бот + Mini App |
|---|---|---|
| UX | Чат-интерфейс: текст → кнопка → текст | Привычный мобильный интерфейс с вкладками |
| Навигация | Линейная, через меню | Свободная, как в приложении |
| Информационная плотность | 1 экран = 1 сообщение | 1 экран = вся нужная информация |
| Персонализация UI | Ограничена | Полная: цвета, анимации, фонты |
| Конкурентное сравнение | Уступает Co-Star / Moonly | Сравнимо с нативным приложением |

### Архитектура Selenyx с Mini App

```
Telegram (бот)
├── Уведомления → openTelegramLink → Mini App
├── /start, онбординг → Mini App (выбор знака)
└── Menu → Mini App (основной интерфейс)

Mini App (webapp/index.html — SPA)
├── Сплэш + анимация Луны
├── ✨ Мой день   → GET /api/today
├── 🌙 Луна       → GET /api/moon
├── 🌟 Карта      → GET /api/natal
└── 💞 Совместимость → GET /api/compat

REST API (api.py — aiohttp)
├── Аутентификация: HMAC Telegram initData (все /api/*)
├── Данные: из astro.py (kerykeion) + data.py
└── БД: SQLite через db.py
```

**Ключевое решение:** бот остаётся только для уведомлений и deep links. Весь пользовательский контент — в Mini App. Это стандартная модель коммерческих ботов (2025).

### Безопасность (initData HMAC)

Каждый запрос к `/api/*` проверяется по стандарту Telegram:

```
secret = HMAC-SHA256("WebAppData", bot_token)
hash   = HMAC-SHA256(data_check_string, secret)
```

Если подпись не совпадает → `401 Unauthorized`. Это защищает данные пользователей и предотвращает несанкционированный доступ к API.

### Monetization через Mini App (Telegram Stars)

Mini App открывает более удобный путь к монетизации:

- **In-app purchase flow:** пользователь видит контент → нажимает «разблокировать» → Telegram Stars — не выходя из приложения
- **Premium gates:** закрытый контент с blur-эффектом виден (создаёт желание), но заблокирован
- **Целевые покупки:** «Прогноз на год — ⭐ 990 Stars», «Детальная совместимость — ⭐ 490 Stars»

Сравнение с нативными приложениями: нет App Store комиссии 30% за покупки Telegram Stars (только Telegram 30%, но нет двойной комиссии).

### UX-поток (реализован в Selenyx)

```
1. Пользователь открывает Mini App (кнопка Menu в боте)
2. Сплэш 500ms → анимация Луны

6. Premium gate на закрытом контенте → ⭐ кнопка Stars
```

Анимация частиц написана на Canvas API (100 частиц, физика: гравитация 0.18, трение 0.97). Spring-анимация карточек: `cubic-bezier(0.34, 1.56, 0.64, 1)` с задержкой 90ms между карточками.

### Ограничения и особенности

- **initData** доступен только в настоящем Telegram-клиенте (не в браузере) → тестировать только через бот
- **tg.initData** пустой при локальном открытии → API отдаёт 401 → Mini App показывает ошибку (ожидаемое поведение)
- **Шрифты из CDN** (Google Fonts): при первом открытии без кэша — небольшая задержка рендера
- **Безопасные зоны** (`env(safe-area-inset-bottom)`) обязательны для iPhone с notch/island
- **Telegram не кэширует** Mini App агрессивно — изменения в index.html появляются быстро
