> Создан: 2026-04-02

## Краткое описание

Лендинг-витрина для новых пользователей Selenyx. Человек попадает из рекламы на `selenyx.ru`, понимает что это за продукт, получает персонализированный вкус прогноза по своему знаку и переходит в Telegram-бот.

**Цель одна:** конвертировать посетителя рекламы → в пользователя бота.

---

## Контекст

- Сейчас `github.io/selenyx` открывает сразу Mini App — без объяснений что это
- Новый пользователь из рекламы не понимает куда попал и уходит
- Нужна отдельная страница-витрина перед входом

## UX-решения

### Интерактив — принцип Saturn
- Пользователь выбирает знак → получает реальный контент (не заглушку)
- Данные берём из того же `data.js` — актуальный прогноз на сегодня
- Выбранный знак сохраняем в localStorage → передаём в Mini App через URL-параметр `?sign=leo`

### Связь с Mini App
- Кнопка ведёт на `t.me/Selenyx_mybot` 
- Если пользователь уже в Telegram — сразу открывается бот
- Опционально: кнопка «Открыть Mini App» → прямая ссылка `t.me/Selenyx_mybot/app`

### Адаптивность
- Mobile-first (большинство трафика с телефонов)
- На мобильном знаки — горизонтальный скролл или 4×3 сетка
- Кнопка CTA — фиксированная внизу экрана на мобильном

## Что НЕ входит в лендинг

- Цены / paywall (это в Mini App)
- Регистрация / форма email (в Mini App после 3-го дня)
- Натальная карта (платная функция)
- Полный прогноз (только вкус — 3-4 строки)

## Технический план реализации

### Шаг 1 — Создать landing/ директорию в репозитории
```
/landing/
  index.html
  style.css
  app.js        ← использует data.js из ../tg-app/data.js (или копия)
```

### Шаг 2 — Настроить nginx на Beget
```nginx
server {
    listen 80;
    server_name selenyx.ru www.selenyx.ru;
    root /home/selenyx/landing;
    index index.html;
}
```
+ certbot для SSL.

### Шаг 3 — Деплой
```bash
# При каждом обновлении лендинга:
ssh root@45.9.43.149
cd /home/selenyx/app && git pull
cp -r landing/* /home/selenyx/landing/
```

### Шаг 4 — Настроить DNS
В панели регистратора selenyx.ru:
- A-запись: `@` → `45.9.43.149`
- A-запись: `www` → `45.9.43.149`
