## 1. Концепция и Логика

### Роль бота

**Режим:** Ежедневный ритуал + Ситуативный советник.

Бот — не оракул и не гороскоп «для всех». Это персональный инструмент синхронизации с лунным ритмом. Язык — психологический: не «сегодня будет…», а «хорошее время для…».

Ключевая метафора: **компас, не расписание**. Показывает направление — пользователь решает сам.

### User Flow

```
/start
  │
  ├── [Новый пользователь]
  │     └── Онбординг → выбор знака → первый прогноз → CTA подписки на уведомления
  │
  └── [Вернувшийся пользователь]
        └── Главное меню
              │
              ├── ✨ Мой день ──────────────────────────────────────────────┐
              │     └── Intro фазы + знак Луны + лунный день               │
              │           └── [Выбор вкладки]                              │
              │                 ├── ✅ Хорошо → детали + кнопка предсказания│
              │                 └── 🚫 Лучше отложить → детали + кнопка     │
              │                       └── 🥠 Открыть предсказание → spoiler ┘
              │
              ├── ✏️ Сменить знак → список знаков → подтверждение
              │
              └── ℹ️ О боте → описание + дисклеймер + контакты
```

**Сценарий «Утренний ритуал»** (основной, 70% трафика):
Уведомление 8:00 → открыть бот → «✨ Мой день» → выбрать вкладку → прочитать → поделиться скрином.

**Сценарий «Ситуативная проверка»** (30% трафика):
Перед встречей / важным решением → `/today` или кнопка → мгновенный ответ → закрыть бот.

## 2. Синтез рекомендаций трёх экспертов

### CUX-Архитектор: логика диалога

**Проблема конкурентов → наше решение:**

| Боль (из отзывов) | Конкурент | Решение в Selenyx |
|---|---|---|
| Агрессивный paywall | Moonly, Co-Star | Весь базовый контент бесплатен |
| Много уведомлений | Все | Максимум 1 в день, время — выбор пользователя |
| Общие тексты | Все | 96 персональных шаблонов знак × фаза |
| Сложный онбординг | Saturn.Love | Только знак, без даты/места рождения |
| Ответ > 3 секунд | Все | Все данные вычисляются < 1 сек |

**Tone of Voice:**
- Тёплый, поддерживающий — не пугающий
- Не «ты должен», а «хорошее время для»
- Не «произойдёт», а «поддерживает», «помогает»
- Уважение к выбору: вкладки — пользователь сам решает что читать

**Предотвращение тупиков:**
- После каждого экрана — кнопка возврата в главное меню
- Catch-all хендлер: если бот не понял — предлагает главное меню
- После открытия предсказания (spoiler) — появляется кнопка «← Назад»

**Скрипт отработки возражения «это не работает»:**
```
Я не предсказываю будущее — я помогаю замечать ритм.
Лунные циклы влияют на сон и биоритмы (это научный факт).
Пробуй неделю — сам почувствуешь разницу.
```

---

### Бот-Дизайнер: интерфейс

**Два типа клавиатур:**

`ReplyKeyboardMarkup` (нижняя, постоянная) — главное меню:
```
   ✨ Мой день
✏️ Сменить знак   ℹ️ О боте
```

`InlineKeyboardMarkup` — контекстные действия внутри сообщений:
- Выбор знака зодиака в онбординге
- Вкладки ✅/🚫 в «Моём дне»
- Кнопки «← Назад» и «🥠 Открыть предсказание»
- CTA «🔔 Включить уведомления» (Шаг 8)

**Медиа:**
- Эмодзи фаз Луны как визуальные маркеры: 🌑🌒🌓🌔🌕🌖🌗🌘
- Стикер при первом запуске (опционально) — приветствие
- Анимированный аватар бота (BotFather → `.tgs` файл, стиль ночного неба)

**Карточка дня — структура сообщения:**
```
🌕 Растущая Луна · Луна в Деве · 11 лунный день

[2-3 строки intro энергетики]

✅ Хорошо сегодня:
· пункт 1
· пункт 2
· пункт 3

[← Назад]  [🥠 Открыть личное предсказание →]
```

### Backend-Разработчик: архитектура

**Критичные требования производительности:**
- Ответ на нажатие кнопки — < 1 сек (астро-расчёты кэшировать на 1 час)
- Кэш астро-данных — `dict` в памяти, ключ = дата YYYY-MM-DD + TZ
- SQLite достаточен до 10 000 пользователей — PostgreSQL при масштабировании
- Long polling на старте, webhook после нагрузки > 1 000 RPS

**FSM (Finite State Machine) состояния:**
```python
class UserState(StatesGroup):
    choosing_sign    # онбординг: ждём выбора знака
    changing_sign    # смена знака: ждём нового выбора
    # остальные взаимодействия — через callback_query без FSM
```

## 3. Структура меню и команд

### Команды бота

| Команда | Действие |
|---|---|
| `/start` | Онбординг (новый) / главное меню (вернувшийся) |
| `/today` | Энергия дня — экран «✨ Мой день» |
| `/moon` | Положение Луны сейчас |
| `/help` | Справка + контакты |
| `/sign` | Сменить знак зодиака |

### Карта клавиатур

```
ГЛАВНОЕ МЕНЮ (ReplyKeyboard)
├── ✨ Мой день
│     └── [Inline] ✅ Хорошо | 🚫 Лучше отложить
│           └── [Inline] ← Назад | 🥠 Открыть предсказание
│                 └── spoiler в том же сообщении, кнопки исчезают
├── ✏️ Сменить знак
│     └── [Inline] сетка 2×6 знаков → подтверждение
└── ℹ️ О боте
      └── [Inline] 📬 Написать автору
```

### FSM — переходы состояний

```
[любое состояние]
      │
      ├── /start → проверка DB
      │     ├── новый → OnboardingStart → choosing_sign → MainMenu
      │     └── есть знак → MainMenu
      │
      ├── "✨ Мой день" → вычислить астро → показать intro
      │     ├── callback "tab_good" → edit message (✅ вкладка)
      │     ├── callback "tab_avoid" → edit message (🚫 вкладка)
      │     ├── callback "open_cookie" → добавить spoiler, убрать кнопки
      │     └── callback "back" → вернуться к intro
      │
      ├── "✏️ Сменить знак" → changing_sign → выбор → сохранить → подтверждение
      │
      └── [неизвестный текст] → catch_all → показать главное меню
```

## 5. Визуальный стиль в чате

### Форматирование HTML

```html
<!-- Заголовок экрана -->
🌕 <b>Полнолуние · Луна в Рыбах · 14 лунный день</b>

<!-- Метки-пункты -->
· Пункт первый
· Пункт второй

<!-- Персональный совет -->
<b>♈ Для Овна:</b> текст совета

<!-- Предсказание -->
🥠 <b>Твоё предсказание:</b>
<tg-spoiler>скрытый текст</tg-spoiler>

<!-- Дисклеймер -->
<i>курсив для юридического текста</i>

<!-- Код — не используется в пользовательских сообщениях -->
```

### Система эмодзи

| Элемент | Эмодзи | Назначение |
|---|---|---|
| Фазы Луны | 🌑🌒🌓🌔🌕🌖🌗🌘 | Визуальный маркер фазы |
| Хорошо | ✅ | Вкладка положительных рекомендаций |
| Отложить | 🚫 | Вкладка ограничений |
| Предсказание | 🥠 | Fortune cookie |
| Главная кнопка | ✨ | «Мой день» в меню |
| Смена знака | ✏️ | Редактирование профиля |
| О боте | ℹ️ | Справка |
| Знаки зодиака | ♈♉♊♋♌♍♎♏♐♑♒♓ | Онбординг и подтверждение |

### Правила форматирования

- Максимум 1 эмодзи в заголовке экрана — не повторять
- Пункты списка: `·` (точка-пуля), не дефис и не цифра
- Двойной `\n\n` между смысловыми блоками
- Никаких `━━━━━━━━` разделителей
- Длина одного сообщения — умещается без скролла на среднем телефоне

### Схема базы данных

```sql
CREATE TABLE users (
    user_id     INTEGER PRIMARY KEY,  -- Telegram user_id
    username    TEXT,
    first_name  TEXT,
    zodiac_sign TEXT,                 -- 'aries', 'taurus', ...
    notify_time TEXT,                 -- '08:00' (Шаг 8)
    notify_tz   TEXT DEFAULT 'Europe/Moscow',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### Интеграции

**Текущие:**
- Telegram Bot API — long polling, `ParseMode.HTML`
- kerykeion + Swiss Ephemeris — расчёты фазы, знака, лунного дня

**Планируемые (Шаг 8):**
- APScheduler — рассылка в заданное время
- Timezone-aware уведомления через `zoneinfo`

**Опциональные (при росте):**
- OpenAI API — GPT-генерация текстов при базе 500+ пользователей
- Google Sheets / Notion — сбор обратной связи из тестов

### Кэширование астро-данных

```python
# Кэш на уровне приложения — пересчёт раз в час
_astro_cache: dict = {}  # ключ: "YYYY-MM-DD"

async def get_today_data() -> AstroData:
    key = datetime.now(MOSCOW_TZ).strftime("%Y-%m-%d")
    if key not in _astro_cache:
        _astro_cache[key] = calculate_astro(key)
    return _astro_cache[key]
```

### Обработка ошибок

| Ситуация | Поведение бота |
|---|---|
| Неизвестный текст | Catch-all → «Не понял 🤔 Используй кнопки ↓» + кнопка «✨ Мой день» |
| Ошибка расчётов kerykeion | Лог ошибки + сообщение «Что-то пошло не так. Попробуй через минуту 🌙» |
| Пользователь без знака | Redirect на онбординг выбора знака |
| Telegram API timeout | Автоповтор 3 раза с backoff, затем — молчание |
| Падение бота | Railway автоперезапуск + уведомление разработчику (опционально — Telegram-канал логов) |

## 7. Метрики и тестирование

### Целевые действия и конверсии

| Метрика | Цель | Критично |
|---|---|---|
| Онбординг completion | > 80% выбирают знак | Да |
| Нажатие «✨ Мой день» | > 60% от открывших бот | Да |
| Открытие вкладки | > 50% от вошедших в экран | Нет |
| Открытие предсказания | > 30% | Нет |
| Retention D1 | > 30% | Да |
| Retention D7 | > 15% | Нет |
| Время ответа | < 3 секунды | Да |

### Чек-лист тестирования сценариев

**Онбординг:**
- [ ] Новый пользователь — видит приветствие и кнопку без прокрутки
- [ ] Выбор знака — сохраняется в БД
- [ ] После выбора — сразу открывается «Мой день»
- [ ] Повторный `/start` — не запускает онбординг заново

**«Мой день»:**
- [ ] Фаза, знак, лунный день — отображаются корректно
- [ ] Кнопки ✅/🚫 редактируют то же сообщение (не создают новое)
- [ ] Персональный совет соответствует знаку пользователя
- [ ] Предсказание появляется как `<tg-spoiler>` в том же сообщении
- [ ] После открытия предсказания кнопки исчезают
- [ ] «← Назад» возвращает к intro экрану

**Навигация:**
- [ ] «✏️ Сменить знак» — знак обновляется, следующий прогноз учитывает новый
- [ ] Случайный текст → catch-all → предлагает главное меню
- [ ] `/help` — выводит справку с кнопкой «📬 Написать автору»

**Производительность:**
- [ ] Ответ на нажатие кнопки — < 1 секунды
- [ ] Расчёт астро-данных — < 500 мс (проверить с кэшем и без)
- [ ] Бот работает корректно после 24 часов непрерывной работы

**Граничные случаи:**
- [ ] Пользователь нажимает кнопку несколько раз подряд — нет дублей сообщений
- [ ] Пользователь из другого часового пояса — дата и лунный день корректные
- [ ] База SQLite не повреждается при одновременных запросах
