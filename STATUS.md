### Платная совместимость — план реализации

**Что есть сейчас (бесплатный тизер):**
- 11 текстовых комбинаций по стихиям (огонь/земля/воздух/вода)
- Пример: Овен + Лев = «Огонь + Огонь» → один фиксированный текст
- Кнопка «Поделиться» для виральности
- Хранится в `data.js` → `_COMPAT_TABLE`

**Что нужно построить (платная синастрия):**

Синастрия — это сравнение двух натальных карт. Kerykeion уже умеет считать натальные карты (`get_natal_chart` в `astro.py`). Нужно:

1. **Собрать данные партнёра**
   - Попросить пользователя ввести дату рождения партнёра (и время, опционально)
   - Хранить в `users.partner_birth_date`, `users.partner_birth_time` (новые колонки)

2. **Рассчитать обе карты**
   ```python
   user_chart    = get_natal_chart(user.birth_date, user.birth_time)
   partner_chart = get_natal_chart(partner_birth_date, partner_birth_time)
   ```

3. **Сравнить ключевые планеты** (для MVP синастрии достаточно 5 пар):
   | Пара | Что значит |
   |------|-----------|
   | Солнце пользователя × Луна партнёра | Эмоциональная совместимость |
   | Луна × Луна | Бытовое взаимопонимание |
   | Солнце × Солнце | Характеры, базовая совместимость |
   | Венера × Марс | Влечение, страсть |
   | Меркурий × Меркурий | Общение, коммуникация |

4. **Контент** — написать тексты для ~60 комбинаций (12 знаков × 5 пар)
   - Тон: «Светлый, без суда» — ни одного «вы несовместимы»
   - Пример: «Твоё Солнце в Овне встречает Луну партнёра в Раке — вы дополняете друг друга: один ведёт, другой держит тепло»

5. **Виральная петля** (Co-Star Bonds-механика):
   - Пользователь отправляет партнёру ссылку `t.me/Selenyx_mybot/app?startapp=pair_TOKEN`
   - Партнёр вводит свою дату → оба получают совместный прогноз
   - Это регистрация нового пользователя через виральную механику

**Файлы которые затронет реализация:**

| Файл | Что изменить |
|------|-------------|
| `db.py` | Добавить `partner_birth_date TEXT`, `partner_birth_time TEXT` в `users` |
| `astro.py` | Новая функция `get_synastry(chart1, chart2)` → список аспектов |
| `api.py` | Новый эндпоинт `POST /api/compat/synastry` + `GET /api/compat/partner` |
| `data.py` | Новый словарь `SYNASTRY_TEXTS` — ~60 комбинаций |
| `tg-app/data.js` | Аналогичные тексты для Mini App (или через API) |
| `tg-app/app.js` | Форма ввода даты партнёра в скрытой вкладке «Пара» |
| `tg-app/index.html` | Раскрыть tab-pane `data-tab="compat"` (уже есть в HTML, скрыта) |

**Зависимости:**
- Требует натальную карту пользователя (должна быть введена заранее)
- Без даты рождения пользователя — синастрия недоступна → стимул заполнить карту

### Натальная карта — план расширения

**Проблема:** сейчас 3 отдельных предложения, склеенных подряд. Пользователь платит 690₽ и видит то, что есть в любом бесплатном гороскопе.

**Что нужно построить:**

1. **`NATAL_SUN_MOON_COMBO`** (data.py) — 144 текста (12 знаков × 12 знаков)
   - Формат: Солнце × Луна как одна история про человека, 2–3 предложения
   - Пример (Овен ☀️ + Рак 🌙): «Снаружи — быстрый и смелый, внутри — глубоко чувствующий и нуждающийся в близости. Ты берёшь на себя больше, чем можешь унести, а потом уходишь восстанавливаться.»
   - В `astro.py get_natal_chart()` добавить: `"combo": NATAL_SUN_MOON_COMBO.get((sun_key, moon_key), "")`

2. **Расширенные описания** (data.py) — 4 блока вместо 1 предложения:
   - Суть → Тень → Что заряжает → Совет

3. **Транзит-хинт** (astro.py) — новая функция `get_natal_transit_hint(natal_moon, transit_moon)`:
   - Если транзитная Луна = натальная Луна → «Луна в твоём знаке — особый день»
   - Добавляется в «Мой день» как дополнительная строка

4. **Число судьбы + знак** — уже считается в `get_life_path_number()`, добавить комбо-текст

**UX-journey для вовлечения (3 шага):**

```
Без данных:    [Тизер-карточка с размытым текстом] → кнопка «Узнать своё ядро»
После даты:    reveal ☀️ Солнце + 🌙 Луна + 🔢 Число + combo-текст
               → кнопка «Добавь время → разблокируй Асцендент»
После времени: полный портрет ☀️ + 🌙 + ↑ Асцендент + расширенные тексты
```

**Файлы:**
| Файл | Изменение |
|------|-----------|
| `data.py` | `NATAL_SUN_MOON_COMBO` ~144 текста + расширить 3 словаря до 4 блоков |
| `astro.py` | `get_natal_chart()` +1 поле combo; новая `get_natal_transit_hint()` |
| `tg-app/app.js` | UX-journey: тизер → форма → animated reveal → запрос времени |
| `tg-app/style.css` | Анимация reveal для портрета |
