## 1. Архитектура

```
Telegram-бот (@Selenyx_mybot)        Railway
┌────────────────────────────────────────────────────┐
│  bot.py   — aiogram хендлеры + APScheduler        │
│  api.py   — aiohttp REST API (порт 8080)           │
│  astro.py — kerykeion (Swiss Ephemeris)            │
│  db.py    — SQLite WAL (/data/selenyx.db)          │
└────────────────────────────────────────────────────┘
                        ↑ не связан ↓
Mini App (GitHub Pages) — автономный SPA, без API
┌────────────────────────────────────────────────────┐
│  tg-app/index.html — HTML-разметка                │
│  tg-app/app.js     — вся логика (~1017 строк)     │
│  tg-app/data.js    — контентные данные (~434 ст.) │
│  tg-app/style.css  — стили (~1028 строк)          │
└────────────────────────────────────────────────────┘
URL: https://anastasiatrusova204-beep.github.io/selenyx/
Прямая ссылка: https://t.me/Selenyx_mybot/app
```

**Важно:**
- Mini App в `tg-app/` НЕ делает запросов к API. Все данные в `data.js`, расчёты в `app.js`.
- Mini App в `webapp/` — устаревшая версия на Railway, пользователи её не видят.
- Бот и API — один процесс (bot.py запускает оба через asyncio).

## 2. Локальный запуск

### 2.1 Требования

```
Python 3.9.6   (команда: python3 --version)
Git            (команда: git --version)
```

### 2.2 Установка зависимостей

```bash
git clone https://github.com/anastasiatrusova204-beep/selenyx.git
cd selenyx
python3 -m venv venv
source venv/bin/activate      # macOS/Linux
# venv\Scripts\activate       # Windows

pip install -r requirements.txt
```

Пакеты в requirements.txt:
```
aiogram==3.13.1
kerykeion==4.26.3
aiosqlite==0.20.0
APScheduler==3.10.4
python-dotenv==1.0.1
aiohttp>=3.9
```

### 2.3 Настройка `.env`

Скопировать `.env.example` в `.env` и заполнить:

```bash
cp .env.example .env
```

| Переменная | Описание | Пример | Обязательна |
|-----------|----------|--------|-------------|
| `BOT_TOKEN` | Токен бота от BotFather | `1234567890:AAABBB...` | ✅ |
| `ADMIN_IDS` | Telegram user_id для /admin/stats | `958560798` | Для admin-тестов |
| `DEMO_MODE` | Отключить HMAC-авторизацию API | `true` | Для тестирования API |
| `DB_PATH` | Путь к БД | не указывать (auto) | ❌ |
| `PORT` | Порт API | `8080` | ❌ |

> ⚠️ Не запускать локальный бот одновременно с Railway — возникнет `TelegramConflictError`.

### 2.4 Запуск бота

```bash
# Убить старый процесс, запустить заново (sleep 6 обязателен)
pkill -9 -f "bot.py" 2>/dev/null; sleep 6
venv/bin/python3 bot.py > bot.log 2>&1 &
sleep 8 && cat bot.log
```

**Признак успешного запуска** в `bot.log`:
```
INFO - Bot started
INFO - Scheduler started
INFO - aiohttp.access - GET /health HTTP/1.1 200
```

### 2.5 Запуск Mini App локально (без Telegram)

Mini App — обычные HTML/JS/CSS файлы. Открыть `tg-app/index.html` в браузере:

```bash
# Вариант 1: через Python HTTP-server
cd tg-app
python3 -m http.server 3000
# Открыть: http://localhost:3000

# Вариант 2: Live Server в VSCode
# Открыть tg-app/index.html → ПКМ → Open with Live Server
```

> ⚠️ Telegram SDK (`tg.initData`, `tg.HapticFeedback`, `tg.BackButton`) не работает вне Telegram.
> В браузере приложение работает как обычная HTML-страница — все функции доступны,
> кроме Telegram-специфичных (haptics, BackButton, initData).

### 2.6 Остановка бота

```bash
pkill -9 -f "bot.py"
```

## 4. Тест: Telegram-бот

### 4.1 Команды

Протестировать каждую команду в чате с @Selenyx_mybot.

#### `/start` — новый пользователь

**Условие:** пользователь не зарегистрирован (новый аккаунт или после `/resetme`)

**Ожидаемый результат:**
```
🌙 Привет, [Имя]!
...текст приветствия...
[кнопка «Начать →»]
[кнопка «Selenyx» — открывает Mini App]
```

**Проверить:**
- [ ] Имя пользователя подставляется правильно
- [ ] Кнопка «Начать →» существует
- [ ] При нажатии «Начать →» → предлагает выбрать знак зодиака
- [ ] Знак сохраняется в БД после выбора

#### `/start` — возвращающийся пользователь

**Условие:** пользователь ранее выбирал знак

**Ожидаемый результат:**
```
🌙 С возвращением, [Имя]!
...текст...
[кнопка «🌙 Перейти в приложение →»]
```

**Проверить:**
- [ ] Бот узнаёт пользователя
- [ ] Кнопка ведёт на https://anastasiatrusova204-beep.github.io/selenyx/

#### `/start wa_moon`, `/start wa_my_day`, `/start wa_natal`, `/start wa_compat`

**Ожидаемый результат:** бот отвечает с кнопкой, после нажатия Mini App открывается на нужной вкладке.

**Проверить:**
- [ ] `wa_moon` → вкладка 🌙
- [ ] `wa_my_day` → вкладка ✨
- [ ] `wa_natal` → вкладка с натальной картой
- [ ] `wa_compat` → вкладка совместимости

#### `/help`

**Ожидаемый результат:** список функций бота с описанием. Кнопка «Перейти в приложение».

**Проверить:**
- [ ] Текст помещается в одно сообщение (без скролла в Telegram)
- [ ] Ссылка на приложение рабочая

#### `/app`

**Ожидаемый результат:** подсказка с кнопкой для открытия Mini App.

#### `/email` — у пользователя нет email

**Ожидаемый результат:**
```
📧 Хочешь получать прогнозы на email?
[«Добавить email»] [«Нет, спасибо»]
```

**Проверить:**
- [ ] При нажатии «Добавить email» бот ждёт ввода
- [ ] При вводе `test@example.com` — сохраняется, подтверждение
- [ ] При вводе `notanemail` — отклоняется, ошибка
- [ ] При вводе `/cancel` — отменяет FSM, возвращает в обычный режим

#### `/email` — у пользователя уже есть email

**Ожидаемый результат:** показывает текущий email. Кнопки «Изменить» / «Удалить».

**Проверить:**
- [ ] «Удалить» → email становится NULL в БД, подтверждение пользователю

#### `/cancel`

**Условие:** бот ждёт ввода (после `/email` или другого FSM-состояния)

**Ожидаемый результат:** «Действие отменено.» — FSM сбрасывается.

**Проверить:**
- [ ] После `/cancel` бот перестаёт ждать ввода
- [ ] Обычные команды снова работают

#### `/resetme`

**Ожидаемый результат:** подтверждение удаления данных, затем их фактическое удаление.

**Проверить:**
- [ ] После `/resetme` → пользователь удаляется из таблицы `users`
- [ ] После `/resetme` → `/start` снова показывает экран нового пользователя

### 4.2 Меню (кнопки)

Проверить все кнопки постоянного меню:

| Кнопка | Ожидаемое действие | Проверить |
|--------|-------------------|-----------|
| ✨ Мой день | Открывает инлайн-кнопки или текст энергии дня | Включает знак Луны, фазу, персонализацию по знаку |
| 📅 Календарь | Лунный календарь на текущий месяц | Фазы совпадают с astro.com |
| 🔔 Уведомления | Предлагает выбрать время | 07:00–11:00, вариант «Отключить» |
| 🌟 Моя карта | Запрашивает дату рождения если нет данных | Или показывает сохранённую карту |
| 💞 Совместимость | Предлагает выбрать знак партнёра | Кнопка «Поделиться» после результата |
| ✏️ Сменить знак | Предлагает выбрать новый знак | После выбора — подтверждение |
| ℹ️ О боте | Информация + FAQ + Словарь | Кнопки «FAQ ❓» и «Словарь 📖» |

### 4.3 Уведомления (APScheduler)

**Условие:** установить уведомление на ближайшее время (через 1–2 минуты от текущего).

**Ожидаемый результат:** в установленное время приходит сообщение с энергией дня.

**Формат уведомления должен включать:**
- Фазу Луны с emoji
- Знак Луны
- Персонализированный совет (зависит от знака пользователя)
- Цвет дня

**Проверить:**
- [ ] Уведомление приходит без открытия бота
- [ ] Текст отличается от вчерашнего (если фаза Луны изменилась)
- [ ] После 7 дней подряд — приходит «Итог недели»

### 4.4 FSM-состояния

**Таблица состояний:**

| Состояние (из bot.py) | Триггер | Выход | Тест |
|-----------------------|---------|-------|------|
| `choosing_sign` | Нажатие «Начать →» у нового пользователя | Выбор знака или `/cancel` | Выбрать Овна → знак сохраняется |
| `changing_sign` | Кнопка «✏️ Сменить знак» | Выбор нового знака или `/cancel` | Сменить с Овна на Льва |
| `entering_birth_date` | Кнопка «🌟 Моя карта» без данных | Ввод даты `ДД.ММ.ГГГГ` или `/cancel` | Ввести `21.03.1990` |
| `entering_birth_time` | После успешного ввода даты | Ввод `ЧЧ:ММ` или «Пропустить» | Ввести `14:30` или нажать «Пропустить» |
| `entering_email` | Кнопка «Добавить email» | Ввод email или `/cancel` | Ввести `user@example.com` |

## 5. Тест: Mini App

> Тестировать в Telegram Mobile (iOS или Android) — только там работает Telegram SDK.
> Для браузерного тестирования: открыть `http://localhost:3000`, функции без SDK будут работать.

### 5.1 Первый визит (нет `userSign` в localStorage)

**Поток:**
```
Сплэш-экран → Онбординг (3 слайда) → Выбор знака → Главный экран
```

**Тесты сплэш-экрана:**
- [ ] Сплэш отображается при загрузке
- [ ] Через ~1.2 сек (или при тапе) → переходит на следующий экран
- [ ] Анимация луны воспроизводится (пульсация `moonPulse`)

**Тесты онбординга:**
- [ ] Слайд 0: приветственное сообщение
- [ ] Слайд 1: описание функций
- [ ] Слайд 2: сетка выбора знака (12 знаков)
- [ ] Кнопка «Далее →» переключает слайды
- [ ] Точки-индикаторы показывают текущий слайд
- [ ] Выбранный знак выделяется (`.active` класс)
- [ ] Кнопка «Начать ✨» деактивирована пока знак не выбран
- [ ] После нажатия «Начать ✨» → `localStorage.setItem('userSign', 'aries')` (или другой)
- [ ] Переход на главный экран

**Негативный тест:**
- [ ] Нажать «Начать ✨» без выбора знака → toast «Выбери свой знак зодиака» + вибрация (в Telegram)

### 5.2 Повторный визит (есть `userSign` в localStorage)

**Поток:**
```
Сплэш-экран → Главный экран (вкладка «День»)
```

**Проверить:**
- [ ] Онбординг НЕ показывается
- [ ] Сразу загружается вкладка «День»
- [ ] Имя пользователя подставляется в заголовок (из `tg.initDataUnsafe.user`)

### 5.3 Вкладка «✨ День»

**Содержимое вкладки (сверху вниз):**

| Элемент | ID/класс | Данные | Проверить |
|---------|----------|--------|-----------|
| Строка «Понедельник, 16 марта» | `#today-date` | Текущая дата | Совпадает с реальной датой |
| Карточка-плитка: Фаза Луны | `.tile` | Emoji + название фазы | 8 возможных фаз |
| Карточка-плитка: Луна в знаке | `.tile` | Emoji знака + название | 12 знаков зодиака |
| Кнопки доменов | `.domain-btn` | 🏥 Здоровье / 💼 Работа / ❤️ Отношения / 🧠 Психология | 4 кнопки |
| Карточка «Цвет дня» | `#today-color` | Название + HEX-цвет | Цвет соответствует плашке |
| Карточка «Число дня» | `#today-number` | Цифра 1–9 + текст | Нумерологический расчёт |

**Тесты вкладки:**
- [ ] Все 4 элемента плитки отображаются
- [ ] Нажатие на домен-кнопку **меняет цвет** (active-класс) и **открывает bottom sheet**
- [ ] В bottom sheet отображается советы по выбранному домену
- [ ] Bottom sheet закрывается кнопкой × или свайпом вниз
- [ ] Нажатие на другой домен — предыдущий теряет active, новый получает
- [ ] Нажатие на «Цвет дня» → bottom sheet с описанием цвета + планетой дня
- [ ] Нажатие на «Число дня» → bottom sheet с нумерологическим значением
- [ ] sessionStorage содержит ключ `td_v2_ГГГГ-ММ-ДД` после первой загрузки

**Проверка персонализации:**
- [ ] Сменить знак в настройках с Овна на Льва → советы в доменах меняются
- [ ] Совет дня включает персональный текст (не общий для всех знаков)

### 5.4 Вкладка «🌙 Луна»

**Содержимое:**

| Элемент | Данные | Проверить |
|---------|--------|-----------|
| Emoji фазы (крупно) | Одна из: 🌑🌒🌓🌔🌕🌖🌗🌘 | Совпадает с astro.com |
| Название фазы | Текст на русском | Соответствует emoji |
| Луна в знаке | «Луна в [Знак]» | Соответствует реальному положению |
| Лунный день | «[N]-й лунный день» | Число 1–30 |
| Карточки планет (аспекты) | Планета + аспект | До 5 аспектов |
| Ретроградные планеты | Блок «⬅️ Ретроград» (если есть) | Показывается только при активном ретрограде |

**Тесты:**
- [ ] Все данные отображаются без «undefined» или «null»
- [ ] Лунный день — целое число 1–30
- [ ] Если нет ретроградов — блок скрыт, не показывает пустой элемент
- [ ] sessionStorage содержит ключ `md_v2_ГГГГ-ММ-ДД`

### 5.5 Вкладка «📅 Месяц»

**Содержимое:**

| Элемент | Источник | Проверить |
|---------|----------|-----------|
| Заголовок «Март 2026 ›» | Текущий месяц | Совпадает с реальной датой |
| Кнопки «‹» / «›» | Навигация | Меняют месяц, сетка перерисовывается |
| Баннер K-индекса | NOAA API (localStorage `kindex_ГГГГ-ММ-ДД`) | Цветная точка + текст |
| Переключатель «Календарь» / «График» | — | Скрывает/показывает cal-grid/cal-chart |
| Сетка дней (7 колонок) | calcMoonData(date) | Emoji фазы в каждой ячейке |
| Сегодняшний день | — | Выделен рамкой (класс `.today`) |

**Тесты K-индекса:**
- [ ] При первом открытии вкладки → `fetch` к NOAA API отправляется (Network tab в DevTools)
- [ ] После загрузки → баннер показывает текущий уровень
- [ ] `localStorage.getItem('kindex_2026-03-16')` содержит числовое значение
- [ ] При отключённом интернете → баннер скрыт (silent fail), приложение работает

**Тесты сетки:**
- [ ] Каждый день показывает emoji фазы Луны (🌑🌒🌓🌔🌕🌖🌗🌘)
- [ ] Нажатие на день → открывает bottom sheet «Шторка дня»
- [ ] Шторка содержит: дата, фаза, знак Луны, лунный день, K-индекс, ползунок самочувствия
- [ ] Ползунок 1–10 меняет значение в `#mood-val`
- [ ] Нажать «Сохранить» → ячейка окрашивается в цвет по настроению
- [ ] `localStorage.getItem('calMoods_2026-03')` содержит `{"16": 7}` (или текущее значение)
- [ ] Кнопки ‹/› меняют месяц, данные настроения за разные месяцы не перемешиваются

**Тесты графика:**
- [ ] Нажать «График» → SVG bar chart отображается
- [ ] Бары с данными настроения цветные, без данных — серые
- [ ] Над барами — emoji фазы Луны
- [ ] «Календарь» → возврат к сетке

### 5.6 Вкладка «🥠 Оракул»

**Содержимое:** анимированная иконка печенья, кнопка «разбить».

**Тесты:**
- [ ] Иконка покачивается (анимация `cookieSway`)
- [ ] Нажатие → анимация `cookiePop`, печенька «ломается»
- [ ] Появляется случайный текст предсказания
- [ ] Предсказание не пустое и не «undefined»
- [ ] При повторном открытии вкладки — новое предсказание (не кэшируется)

## 7. Тест: База данных

```bash

```

### 7.1 Структура таблиц

```sql
-- Проверить схему
.schema users
.schema event_log
```

**Ожидаемые колонки `users`:**
```
user_id, first_name, zodiac_sign, created_at, last_visit,
streak, notify_time, birth_date, birth_time, tier, trial_start, email
```

**Если какой-то колонки нет** → migration не прошла, проверить `db.py` функцию `init_db()`.

### 7.2 Проверка данных

```sql
-- Количество пользователей
SELECT COUNT(*) FROM users;

-- Пользователи со знаком
SELECT COUNT(*) FROM users WHERE zodiac_sign IS NOT NULL;

-- Пользователи с email
SELECT COUNT(*) FROM users WHERE email IS NOT NULL;

-- Активность за последние 7 дней
SELECT COUNT(DISTINCT user_id) FROM event_log
WHERE ts >= datetime('now', '-7 days');

-- Топ событий
SELECT event, COUNT(*) FROM event_log GROUP BY event ORDER BY 2 DESC;

-- Стрик конкретного пользователя
SELECT user_id, streak, last_visit FROM users WHERE user_id = 999999999;
```

### 7.3 Проверка streak

**Условие:** пользователь открывает «Мой день» два дня подряд.

```sql
-- До второго дня
SELECT streak FROM users WHERE user_id = [ID]; -- должно быть 1

-- После второго дня
SELECT streak FROM users WHERE user_id = [ID]; -- должно быть 2
```

**Негативный тест:** пропустить один день → streak сбрасывается в 0.

### 7.4 Проверка trial

```sql
SELECT user_id, tier, trial_start,
  julianday('now') - julianday(trial_start) AS days_used
FROM users WHERE user_id = 999999999;
-- days_used должно быть ≤ 7 для нового пользователя
```
