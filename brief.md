## РАЗДЕЛ 0 — СУТЬ ДОКУМЕНТА

Этот файл — главное ТЗ для разработки лендинга Selenyx. Лендинг — внешняя страница (не Mini App), цель которой: привлечь нового пользователя и отправить его в Telegram-бота. Документ описывает структуру, тексты, дизайн и все интерактивные элементы.

---

## РАЗДЕЛ 1 — ЦЕЛЬ И АУДИТОРИЯ

### Цель лендинга

**Единственная цель:** конвертировать посетителя в пользователя Telegram-бота @Selenyx_mybot.

Действие-цель: клик по кнопке → переход в Telegram → `/start` → Mini App.

### Целевая аудитория

**Основная:** женщины 25–40 лет, Россия и СНГ
- Интересуются саморазвитием, психологией, lifestyle
- Скептичны к «астрологии» как слову, но открыты к «ритму», «фокусу», «ясности»
- Пробовали Co-Star или Moonly, ушли из-за агрессивной монетизации
- Привыкли к качественным мобильным приложениям

**Вторичная:** мужчины 28–42 лет
- Ситуативное использование (важные переговоры, решения)
- Ценят конкретность и скорость (не «предсказания», а «компас»)

### Боли аудитории (формулировки из RESEARCH.md §9)

| Боль пользователя | Как говорит | Наш ответ |
|---|---|---|
| Неясность | «Не знаю за что взяться» | «Знай, на что направить энергию» |
| Тревога | «Нет уверенности в решениях» | «Понимай ритм вместо предсказаний» |
| Скептицизм | «Это просто гороскопы» | «Реальные астрономические данные. Swiss Ephemeris (NASA JPL)» |
| Барьер входа | «Слишком дорого и сложно» | «Без регистрации · Без кредитки · 30 секунд» |

## РАЗДЕЛ 2 — КЛЮЧЕВОЕ ПОСЛАНИЕ

### Главный оффер (формула RESEARCH.md §11, Эксперт 2)

```
[БОЛЬ] + [РЕШЕНИЕ] + [СКОРОСТЬ] + [БЕЗ РИСКА]

«Бывает: утро началось, а ясности нет.
Selenyx за 30 секунд показывает,
что поддерживает тебя сегодня.
Реальные астрономические данные. Без терминологии.»
```

### Философский якорь бренда

**«Компас, не расписание»** — Selenyx не диктует и не предсказывает. Показывает ритм, решение принимает пользователь.

### Запрещённые слова

- «предсказание», «судьба», «магия»
- «астрология» (в hero-блоке — заменять на «лунный навигатор», «компас», «ритм дня»)
- «Луна говорит, что…» → мистицизм
- «Избегайте конфликтов» → приказ
- «Сегодня произойдёт…» → фатализм

### Разрешённые слова

- «хорошее время для…», «поддерживает», «компас»
- «30 секунд», «персональный», «синхронизируй», «слушай»
- «обратите внимание», «помогает понять», «ритм»

### Scroll-Reveal для блоков

```javascript
// IntersectionObserver + двойной rAF (паттерн из webapp/index.html)
const observer = new IntersectionObserver(entries => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      requestAnimationFrame(() => {
        requestAnimationFrame(() => {
          entry.target.classList.add('visible');
        });
      });
      observer.unobserve(entry.target);
    }
  });
}, { threshold: 0.1 });

document.querySelectorAll('.reveal').forEach(el => observer.observe(el));
```

```css
.reveal {

  transform: translateY(24px);

}

  transform: none;
}
```

### Performance — обязательные требования

- Интерактивные элементы: keyboard navigation (Tab + Enter)
- Цветовой контраст: минимум 4.5:1 для текста

- Минимальный touch-target: 44×44px
- Safe areas: `padding-bottom: env(safe-area-inset-bottom)`

### Обязательно для каждого экрана

### Принцип «приложение, не сайт»

- Нет скролла всей страницы — только внутри вкладок

- Нет внешних ссылок через `window.open` — только `tg.openLink()`
- Нет browser alert/confirm — только `tg.showPopup()` / `tg.showAlert()`
- Нет горизонтальных отступов > 16px — контент занимает экран
- Таб-бар всегда виден (нижний, фиксированный)
- `tg.expand()` при старте — полный экран, не половина

## РАЗДЕЛ 5 — ЧЕГО НЕ БУДЕТ В ПЕРВОЙ ВЕРСИИ

| Что | Почему нет | Когда |
|---|---|---|
| Покупка через Telegram Stars | Не валидирован спрос | Шаг 12 (после 100+ MAU) |
| Прогноз на неделю/месяц/год | Требует генерацию контента | После монетизации |
| Транзиты планет с уведомлениями | Сложно технически + нет спроса | После MVP |
| История прогнозов (архив) | Мало ценности без retention | Позже |
| Оффлайн-режим | Service Workers в TMA не поддерживаются | Невозможно |
| Push через TMA | Нет доступа без нативного приложения | Всегда через бота |
| Регистрация / пароль | Авторизация через Telegram — уже есть | Не нужно |
| Аффирмации и ритуалы | Низкий приоритет | Контент-итерация |
| Реферальная программа | Сложная логика | После набора аудитории |
| Смена языка (EN) | Аудитория RU/CIS | После масштабирования |
| Социальная лента (Bonds как Co-Star) | Отдельный продукт | Не в roadmap |
| Видео-контент | Нет инфраструктуры | Не в roadmap |

## РАЗДЕЛ 6 — ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ MINI APP

### TMA API — обязательные вызовы

```javascript
// При старте (в init())
tg.ready();           // сигнал: приложение готово к показу
tg.expand();          // полный экран, а не половина

// При сохранении данных
tg.HapticFeedback.notificationOccurred('success');
tg.showPopup({ message: 'Сохранено' });

// При выборе (знак, домен, вкладка)
tg.HapticFeedback.impactOccurred('light');     // лёгкий тап
tg.HapticFeedback.selectionChanged();           // смена выбора

// При деструктивных действиях
tg.showConfirm('Удалить данные?', callback);

// Внешние ссылки — ТОЛЬКО так
tg.openLink('https://...');                    // внешний браузер
tg.openTelegramLink('https://t.me/...');       // внутри Telegram

// BackButton — только для Fortune Cookie
tg.BackButton.show();
tg.BackButton.onClick(() => closeFortune());
// При закрытии fortune:
tg.BackButton.offClick(handler);
tg.BackButton.hide();
```

```javascript
// Все /api/* запросы:
fetch('/api/today', {
```
